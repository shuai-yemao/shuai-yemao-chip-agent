# 嵌入式开发专属约束规则

## 芯片与平台
- 指定 MCU 型号（如 STM32F411CEU6）后，外设驱动按对应参考手册实现
- 默认使用 STM32 HAL 库；明确要求时才使用 LL/寄存器操作
- 优先使用嵌入式技能包（i2c-bus, spi-bus, uart-module 等）实现模块化

## 外设驱动
- I2C: 400kHz 标准速率，支持多设备总线，提供错误检测
- SPI: 按设备规格配置模式（CPOL/CPHA）和速率
- UART: 115200 默认波特率，支持中断/DMA 模式
- 所有外设驱动提供初始化/反初始化/读写三件套

## 固件构建与烧录
- 使用 OpenOCD 或 JLink 烧录，遵循目标芯片的烧录协议
- 编译产物检查：.elf / .hex / .bin 至少保留一种
- 烧录后由用户确认功能正常，单元测试通过 ≠ 硬件验证通过
- 使用 `build-platformio` / `build-cmake` 等技能包构建

## 资源限制
- 注意 Flash 和 RAM 容量限制（STM32F411CEU6: 512KB Flash, 128KB RAM）
- 避免动态内存分配（malloc）在中断/实时路径中使用
- RTOS 任务栈大小按实际使用量配置，预留 20% 余量
