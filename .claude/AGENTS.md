## 通信协议

- **Human-in-the-loop**：关键决策前必须问（删除文件、新依赖、git push）
- **状态透明**：当前在 Alignment Flow 哪个 Step 要说清楚

- **失败上报**：执行失败时给出原因和建议修复方向

## 约束与禁止行为

### 通用约束

- 不修改 `~/.claude/CLAUDE.md` 以外的全局配置
- 不改 `.git/`、`.env` 文件
- 不自动安装系统级依赖（如 apt、brew、pip install --global）
- 不自动执行 git push（需确认）

### 🔧 嵌入式专属约束

- **不修改 MCU 厂商原始文件**（HAL/SPL/CMSIS 核心库）除非有明确理由并经确认
- **不假设寄存器默认值** — 查阅数据手册或头文件确认复位值
- **不删除未知用途的 BSP/驱动代码** — 先确认硬件连接和外设用途
- **不直接向目标设备烧录未经验证的固件** — 先通过模拟器或单元测试验证
- **不忽略 Watchdog 定时器** — 禁用须有明确理由并告知影响
- **不改动 Linker Script 的内存布局** 除非同步更新启动文件和中断向量表
- **不假设编译优化选项的安全性** — volatile、内存对齐、字节序须明确处理
- **不擅自修改 Flash 分区表** — 分区变更影响 OTA 和 Bootloader 兼容性
- **不使用与目标 MCU 不匹配的 Device Family Pack** — 外设寄存器定义可能不一致
- **调试接口操作前确认目标板供电状态** — 调试器可能通过 JTAG/SWD 反向供电

### 🖥️ Ops 层专属约束

- **restore/deploy 必须先 dry-run** — 预览影响范围，确认后再执行
- **不备份/传输 .env.secrets 明文** — 部署时跳过敏感文件，依赖人工配置
- **不覆盖远程已有 skills/ 内容** — 仅同步核心配置和 Workflow
- **prune 保留至少最近 1 个备份** — 无论时间多久，防止意外删除唯一备份
- **doctor 自动修复仅限于安全操作** — chmod、重建索引，不自动修改代码或配置内容
- **分发包不包含 history.jsonl** — 会话历史不纳入分发范围

### 🧰 工具层专属约束

- **不静默升级技能** — 更新必须先显示 diff，经确认后执行
- **卸载前检查反向依赖** — 有被依赖关系时须 force 参数确认
- **安装来源验证** — 仅接受已知可靠来源（GitHub/Gitee），不接受 curl | bash 式安装
- **不破坏 skills 目录结构** — 每个技能保持独立子目录，不混合存放

## 高危操作确认

以下操作必须逐条向用户确认，获得批准后方可执行：

### 文件删除

| 操作 | 确认要求 |
|------|----------|
| 删除单个文件 | 明确文件路径 + 理由 |
| 批量删除 | 列出删除清单 + 总数 + 理由 |
| 递归删除目录 | 列出目录内容概览 + 删除后影响范围 |

### 工具层操作（技能管理）

| 操作 | 确认要求 |
|------|----------|
| 安装技能（install） | 来源 URL + 技能名 + 版本 + 依赖列表 |
| 卸载技能（remove） | 技能名 + 反向依赖检查结果 + 受影响的 layer |
| 更新技能（update） | 技能名 + 当前版本 → 目标版本 + diff 摘要 |
| 强制卸载（force remove） | 技能名 + 被依赖列表 + 理由 |
| 收养技能（adopt） | 技能名 + 路径 + 来源 |

### 🖥️ Ops 层操作（系统运维）

| 操作 | 确认要求 |
|------|----------|
| 全量备份（backup） | 备份范围清单 + 排除项 + 预估大小 |
| 恢复备份（restore） | 备份文件路径 + 文件清单 + dry-run 影响评估（必须先行） |
| 打包分发（package） | 包含内容清单 + 排除项 + 是否含敏感信息 |
| 远程部署（deploy） | 目标地址 + 传输文件清单 + 远程执行命令 + 凭据处理方式 |
| 系统诊断（doctor） | 自动修复项列表（仅限 chmod/重建索引等安全操作） |
| 清理数据（prune） | 清理范围 + 保留天数 + 预估释放空间 |

### 系统命令执行

| 操作 | 确认要求 |
|------|----------|
| 安装/卸载软件包 | 包名 + 版本 + 来源 |
| 修改系统配置 | 配置项 + 原值 + 目标值 |
| 执行网络操作 | 目标地址 + 操作类型 + 数据传输方向 |
| 操作外部进程 | 进程名 + 操作（kill/restart） |

### API 调用

| 操作 | 确认要求 |
|------|----------|
| 第三方 API 调用 | 端点 URL + 请求方法 + 发送的数据字段 |
| 携带认证信息的请求 | 使用的凭据类型 + 权限范围 |
| 写操作 API（POST/PUT/DELETE） | 操作对象 + 数据内容 + 可回滚性 |
| 批量 API 调用 | 调用次数 + 频率 + 总数据量 |

### 🔧 嵌入式专属高危操作

| 操作 | 确认要求 |
|------|----------|
| 烧录/擦除 Flash | 目标芯片 + 起始地址 + 长度 + 影响分区 |
| 修改 Linker Script（`.ld`/`.icf`/`.scf`） | 改前内容 + 改后内容 + 受影响的段 |
| 修改 MCU 时钟配置（PLL/HSE/HSI） | 当前频率 + 目标频率 + 涉及外设 |
| 修改中断向量表 / NVIC 配置 | 改前优先级 + 改后优先级 + 影响的中断线 |
| 修改 Bootloader 分区 | 分区布局 + 回滚方案 |
| 配置 Flash 读保护 / 调试接口锁定 | 操作类型 + 解锁方式（是否可逆） |
| 操作 Option Bytes | 字节地址 + 原值 + 目标值 + 风险说明 |
| 修改 BSP 层 GPIO 定义 | 引脚号 + 复用功能 + 连接的硬件外设 |
| 启用/禁用 Watchdog | 当前状态 + 喂狗周期（禁用须特别说明理由） |
| 执行 OTA 升级流程 | 固件版本 + 升级方式 + 失败回滚方案 |
| 调试器操作（JLink/OpenOCD/ST-Link） | 目标设备 + 操作类型（halt/flash/reset） |

### 命令执行黑名单

| 命令 | 禁止原因 |
|------|----------|
| `rm -rf /`、`rm -rf /*` | 破坏性删除 |
| `dd` 直接写块设备 | 可能破坏磁盘/分区 |
| `chmod -R 777` | 权限过于开放 |
| `> /dev/sda` 等块设备重定向 | 设备破坏 |
| `curl ... \| bash` 管道安装 | 未经审查的远程执行 |
| 🔧 **嵌入式专用** | | |
| `openocd -f ... -c "program ..."` 含未知配置 | 烧录错误配置可能变砖 |
| `jlinkexe` / `JLinkFlash` 含未知设备参数 | 目标芯片参数错误可导致 Flash 损坏 |
| `st-flash` 操作未知起始地址 | Flash 起始地址错误可覆盖关键分区 |
| 直接擦除 `0x08000000` 起始的 Bootloader 区间 | 擦除 Bootloader 且无恢复手段时设备不可启动 |
| 不验证直接写入 Option Bytes | 错误配置可永久锁定调试接口 |

## 审计与异常检测

### Audit Log 记录

| 审计项 | 记录内容 |
|--------|----------|
| 高危操作 | 操作类型 + 时间 + 是否经确认 + 结果 |
| 权限越界尝试 | 目标路径 + 操作 + 拒绝原因 |
| 命令执行 | 完整命令 + exit code + 输出摘要 |
| API 调用 | 端点 + 方法 + 数据量 + 状态码 |
| 文件变更 | 文件路径 + 操作类型 + 变更大小 |

### 定期检查

| 频率 | 操作 |
|------|------|
| 每次对话结束 | 自动检查当前会话的违规行为 |
| 每周 | 全量审查 audit log，标记异常模式 |
| 发现异常 | 立即输出异常报告，询问是否需调整规则 |

### 异常行为识别

以下模式触发告警：

- **高频高危操作**：单位时间内高危操作次数超过阈值
- **权限试探**：连续访问被禁止的路径或资源
- **异常命令链**：正常流程中不会出现的命令组合
- **信息外泄嫌疑**：向未知端点发送代码/数据/凭据
- **规则漂移**：同一类操作越来越频繁地跳过确认步骤

### 异常处理流程

```
发现异常 → 记录到 audit log → 输出异常报告
  ├─ 误报 → 调整规则（更新白名单/阈值）
  ├─ 单次违规 → 确认后修复
  └─ 模式违规 → 更新 AGENTS.md 约束条件
```

## 专用 Agent 行为规范

通过 `claude --agent <name>` 启动的专用 agent 遵守以下规范：

### Agent 权限边界

| Agent | 允许操作 | 禁止操作 |
|-------|---------|---------|
| `embedded-expert` | 读写源码、调用嵌入式技能、执行构建 | 修改 HAL 库、烧录未经验证固件 |
| `code-reviewer` | 读取代码、调用安全层预检、生成报告 | 直接修改代码（仅建议） |
| `test-runner` | 运行测试、读取测试输出、分析结果 | 修改测试文件（仅建议） |
| `devops` | 健康检查、备份、打包、诊断 | 恢复/部署（需主 agent 确认） |

### Agent 与主 Agent 协作

- Agent 完成任务后向主 agent 汇报结果
- Agent 不直接修改全局配置（CLAUDE.md/AGENTS.md/SOUL.md）
- Agent 遇到不确定情况时上报主 agent 决策

## 安全层集成

安全层（safety-layer）为所有操作提供安全护栏：

### 预检机制（Phase 0.5）

```
任务输入 → safety-layer preflight → verdict (allow/caution/block)
  ├─ allow → 直接执行
  ├─ caution → 列出风险，用户确认后执行
  └─ block → 阻止执行，输出原因
```

### 审计机制（Phase 2.5）

```
任务完成 → safety-layer audit → 记录到 audit log
  ├─ 操作类型 + 时间戳
  ├─ 文件变更清单
  ├─ 安全 verdict
  └─ 异常检测结果
```
