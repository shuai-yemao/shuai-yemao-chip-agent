# 记忆系统架构

> 项目知识的多层记忆架构：分层存储、定期总结、混合检索。

## 记忆层次

```
短期记忆 ── 当前会话上下文（对话结束后消亡）
   │
   ↓ 定期总结
   │
近端记忆 ── 最近会话日志与用户交互（可追溯）
   │
   ↓ 提炼+向量化
   │
长期记忆 ── 持久化的重要知识（跨会话保留）
```

### 短期记忆（Working Memory）

| 属性 | 说明 |
|------|------|
| 范围 | 当前对话上下文 |
| 持久化 | 会话结束后消亡 |
| 内容 | 当前任务状态、未完成的决策、临时变量 |
| 管理 | AI 在对话期间自动维护 |

### 近端记忆（Recent Memory）

| 属性 | 说明 |
|------|------|
| 范围 | 近 N 次会话 |
| 存储 | 会话日志（Session Logs） |
| 内容 | 用户交互记录、已完成的决策、关键输出 |
| 维护 | 定期总结→提炼为长期记忆 |

### 长期记忆（Long-term Memory）

| 属性 | 说明 |
|------|------|
| 范围 | 跨所有会话 |
| 存储 | `.claude/memory/` 下的持久文件 |
| 内容 | 重要知识、用户偏好、决策记录、项目目标 |
| 索引 | MEMORY.md（本文件）作为全局索引 |

## 存储机制

### MEMORY.md（记忆索引）

本文件作为全局索引，记录所有记忆的位置和摘要。

当前索引：

| 记忆 | 文件 | 描述 |
|------|------|------|
| 项目目标 | [project-goal.md](.claude/memory/project-goal.md) | 重构四份配置文件的背景与目标 |
| 架构决策 | [arch-decision.md](.claude/memory/arch-decision.md) | 文件结构、引用关系、职责划分 |

### 知识图谱（Knowledge Graph）

> 实体-关系网络，记录项目中的概念、模块、文件及其关联。

- **节点**：文件、模块、概念、决策
- **边**：引用、依赖、实现、继承、影响
- **用途**：推理、影响分析、上下文导航

### 向量库（sqlite-vec）

> 向量化存储，支持语义检索。

- **向量化**：将记忆和知识转化为向量嵌入
- **存储**：sqlite-vec 作为嵌入式向量数据库
- **用途**：语义相似度搜索、模糊匹配

## 核心流程

### 会话日志（Session Logs）

每次对话结束时：
```
对话内容 → 结构化摘要 → 存入近端记忆
                         →（如含重要知识）提炼→ 存入长期记忆
                         →（如涉及新概念）→ 更新知识图谱
                         →（如含可向量化内容）→ 写入向量库
```

### 定期总结

| 频率 | 操作 |
|------|------|
| 每次对话结束 | 提炼新知识、更新决策记录 |
| N 次对话后 | 压缩近端记忆、合并相似条目 |
| 达到阈值 | 触发向量化更新、知识图谱重构 |

### 用户偏好（User Preferences）

- 记录在 `.claude/memory/` 下
- 包含交流语言、输出格式、工具偏好
- 跨会话保持

### 重要知识

识别条件：
- 被重复提及 2+ 次的概念或决策
- 明确的架构决策（含理由和备选方案）
- 项目特有的约定和术语
- 经验教训和已知问题

### 新对话 — 注入上下文

新对话启动时：
```
1. 查询 MEMORY.md 索引 → 获取所有长期记忆
2. 向量库语义检索 → 匹配相关历史知识
3. 知识图谱查询 → 获取当前模块的上下文关系网
4. 混合检索结果 → 注入到对话系统提示中
```

### 混合检索

```
┌─────────────┐    ┌──────────────┐
│ 关键词检索   │    │ 向量语义检索  │
│ (MEMORY.md)  │    │ (sqlite-vec) │
└──────┬───────┘    └──────┬───────┘
       │                   │
       └────────┬──────────┘
                ▼
       ┌────────────────┐
       │  排序 + 去重   │
       └───────┬────────┘
               ▼
       ┌────────────────┐
       │ 注入会话上下文  │
       └────────────────┘
```

## 维护计划

### 每周 Review

| 检查项 | 操作 | 频率 |
|--------|------|:----:|
| 过时条目 | 识别并删除不再适用的记忆 | 每周 |
| 重复条目 | 合并内容重叠的记忆 | 每周 |
| 决策记录 | 确认已有决策是否仍有效 | 每周 |
| 记忆索引 | 检查所有文件路径是否可达 | 每周 |
| 重构日志 | 清理已完成超过 30 天的日志项 | 每月 |

Review 结果记入决策记录表。

### Git 管理

整个 `.claude/memory/` 目录和 `memory.md` 由 **git 管理**：

- **每次变更** → `git add -A && git commit -m "memory: <变更摘要>"`
- **提交信息格式**：`memory: <操作类型> — <内容摘要>`
  - 新增: `memory: add — 项目目标记录`
  - 更新: `memory: update — 决策记录表补充`
  - 删除: `memory: remove — 过期知识条目`
  - 合并: `memory: merge — 重复条目合并`
- **回滚**：通过 `git revert` 或 `git checkout` 恢复旧版本
- **diff 审查**：commit 前 `git diff` 检查变更内容
