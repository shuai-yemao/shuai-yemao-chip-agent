# Agents — AI 行为守则

## 权限等级

| 等级 | 行为 |
|------|------|
| 🟢 **自动** | 读、查、问（不需要确认） |
| 🟡 **确认** | 写、改、删、执行（先问再干） |
| 🔴 **必须你批准** | 不可逆操作（git push、rm -rf、API key 操作等） |

## 文件操作边界

| 路径 | 读 | 写 | 删 | 执行 |
|------|:--:|:--:|:--:|:--:|
| `*.md`（项目根） | 🟢 | 🟡 | 🔴 | — |
| `.claude/` | 🟢 | 🟡 | 🔴 | — |
| `skills/` | 🟢 | 🟡 | 🔴 | — |
| `docs/` | 🟢 | 🟡 | 🔴 | — |
| `.env*` | 🔴 | 🔴 | 🔴 | — |
| `.git/` | 🔴 | 🔴 | 🔴 | 🔴 |
| `build/`、`dist/`、`node_modules/` | 🟢 | 🔴 | 🔴 | — |
| `~/.ssh/` | 🔴 | 🔴 | 🔴 | 🔴 |
| 系统目录（`/etc/`、`/usr/`） | 🔴 | 🔴 | 🔴 | 🔴 |

## 通信协议

- **Human-in-the-loop**：关键决策前必须问（删除文件、新依赖、git push）
- **状态透明**：当前在 Alignment Flow 哪个 Step 要说清楚
- **技能调度**：引用现有技能，不重复造轮子
- **失败上报**：执行失败时给出原因和建议修复方向

## 约束与禁止行为

- 不修改 `~/.claude/CLAUDE.md` 以外的全局配置
- 不改 `.git/`、`.env` 文件
- 不自动安装系统级依赖（如 apt、brew、pip install --global）
- 不自动执行 git push（需确认）

## 高危操作确认

以下操作必须逐条向用户确认，获得批准后方可执行：

### 文件删除

| 操作 | 确认要求 |
|------|----------|
| 删除单个文件 | 明确文件路径 + 理由 |
| 批量删除 | 列出删除清单 + 总数 + 理由 |
| 递归删除目录 | 列出目录内容概览 + 删除后影响范围 |

### 系统命令执行

| 操作 | 确认要求 |
|------|----------|
| 安装/卸载软件包 | 包名 + 版本 + 来源 |
| 修改系统配置 | 配置项 + 原值 + 目标值 |
| 执行网络操作 | 目标地址 + 操作类型 + 数据传输方向 |
| 操作外部进程 | 进程名 + 操作（kill/restart） |

### API 调用

| 操作 | 确认要求 |
|------|----------|
| 第三方 API 调用 | 端点 URL + 请求方法 + 发送的数据字段 |
| 携带认证信息的请求 | 使用的凭据类型 + 权限范围 |
| 写操作 API（POST/PUT/DELETE） | 操作对象 + 数据内容 + 可回滚性 |
| 批量 API 调用 | 调用次数 + 频率 + 总数据量 |

## 权限控制

### 文件访问白名单

| 路径 | 访问级别 | 原因 |
|------|----------|------|
| `项目根目录/*.md` | 读写 | 配置文件维护 |
| `.claude/` | 读写 | 记忆与技能管理 |
| `skills/` | 读写 | 技能文件维护 |
| `docs/` | 读写 | 设计文档与规范 |
| `.git/` | 无访问 | 避免仓库损坏 |
| `.env*` | 无访问 | 凭据保护 |
| `build/`、`dist/`、`node_modules/` | 只读（如需检查产物） | 构建产物，不直接修改 |
| `~/.ssh/` | 无访问 | 密钥保护 |
| 系统目录（`/etc/`、`/usr/` 等） | 无访问 | 系统安全 |

### 命令执行黑名单

| 命令 | 禁止原因 |
|------|----------|
| `rm -rf /`、`rm -rf /*` | 破坏性删除 |
| `dd` 直接写块设备 | 可能破坏磁盘/分区 |
| `chmod -R 777` | 权限过于开放 |
| `> /dev/sda` 等块设备重定向 | 设备破坏 |
| `curl ... \| bash` 管道安装 | 未经审查的远程执行 |

### 网络访问限制

- **允许**：GitHub、GitLab 仓库、文档站点、包注册表（npm/pypi 等）
- **需确认**：未在白名单中的域名、内部服务端点、生产环境 API
- **禁止**：明文传输凭据、向未知端点发送代码/数据

## 资源限制

| 限制项 | 上限 | 说明 |
|--------|:----:|------|
| Token 消耗/会话 | — | 由用户预算决定，超限前预警 |
| 并发任务数 | 4 | 同时执行的任务不能超过 4 个 |
| 单步执行超时 | 300s | 单个命令或操作超过 5 分钟自动中断 |
| 文件写入大小 | 10MB | 单次写入超过 10MB 需确认 |
| 批量操作数 | 50 | 单次批量操作超过 50 条需分拆确认 |

## 隐私保护

### 敏感信息过滤

以下内容在输出和日志中必须自动屏蔽或脱敏：

- API Key、Token、密码、私钥
- 连接字符串中的凭据部分
- 个人身份信息（邮箱、电话、地址）
- 内网 IP / 非公开域名

### 数据脱敏规则

| 数据类型 | 脱敏方式 | 示例 |
|----------|----------|------|
| API Key | 保留首 4 位，其余替换为 `*` | `sk-abc****` |
| 密码 | 完全替换为 `******` | `******` |
| 邮箱 | 用户名首字 + `***@` + 域名 | `z***@example.com` |
| 内网 IP | 保留前两段 | `192.168.*.*` |
