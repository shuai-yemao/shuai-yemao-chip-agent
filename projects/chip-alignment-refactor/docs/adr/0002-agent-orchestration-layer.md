# 0002 — Agent 编排层架构

需求对齐层下方增加编排层，负责调度 + 质量管控。

## Context

需求对齐层产出施工包后，需要有一个层负责任务调度、Agent 管理、质量审查。直接由人工逐个执行效率低下，需要编排层来自动化。

## Decision

编排层采用 4 步流程：解析施工包构建依赖图 → 分批派发 Agent → 执行与重试 → 三层质量管控。

关键设计点：
- **依赖图驱动调度**：按文件/接口/数据依赖构建 DAG，无依赖并行、有依赖串行
- **两层重试**：第 1 次失败自动重试，第 2 次失败停止 + 报警
- **三层门禁**：AI 审查 → 你验证 → 验收清单确认
- **按批次交付**：每批完成后交付 review，再启动下一批
