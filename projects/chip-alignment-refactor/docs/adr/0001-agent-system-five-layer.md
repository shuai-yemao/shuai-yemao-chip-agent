# 0001 — Agent System 五层架构

需求对齐层作为第一层优先构建，其他层后续按依赖顺序加入。

## Context

用户需要将模糊需求（自然语言 + 嵌入式专业术语）转化为结构化方案文档和施工清单。现有技能仓库（grill-me、grill-with-docs、to-issues、to-prd、tdd、diagnose、handoff 等）提供了工程能力但没有编排机制。

## Decision

采用五层 Agent 系统架构，工具层贯通所有层：

1. **需求对齐层** — 模糊需求 → PRD + 施工清单 4 项（当前构建）
2. **Agent 编排层** — 分配工具和任务给各 Agent，tdd 等技能执行
3. **记忆层** — 跨层记忆系统（短期/近端/长期 + 向量库）
4. **安全层** — 跨层权限控制、audit log、异常检测
5. **运维层** — 跨层监控、维护、定期检查

**工具层** 贯通所有层，提供标准化 skills 能力。

构建顺序：需求对齐层优先，其他层按依赖逐步加入。

## Status

accepted
