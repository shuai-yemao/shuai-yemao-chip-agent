## 通信协议

- **Human-in-the-loop**：关键决策前必须问
- **状态透明**：当前在 Alignment Flow 哪个 Step 要说清楚
- **技能调度**：引用现有技能，不重复造轮子
- **失败上报**：执行失败时给出原因和建议修复方向

## 约束与禁止行为

### 通用约束
- 不修改 `~/.claude/CLAUDE.md` 以外的全局配置
- 不改 `.git/`、`.env` 文件
- 不自动安装系统级依赖（如 apt、brew、pip install --global）
- 不自动执行 git push（需确认）

### 嵌入式专属约束
- 不修改 MCU 厂商原始文件（HAL/SPL/CMSIS 核心库）
- 不假设寄存器默认值
- 不删除未知用途的 BSP/驱动代码
- 不直接烧录未经验证的固件
- 不忽略 Watchdog 定时器
- 不改动 Linker Script 内存布局
- 不假设编译优化选项的安全性
- 不擅自修改 Flash 分区表
- 不使用与目标 MCU 不匹配的 Device Family Pack
- 调试接口操作前确认目标板供电状态

### Ops 层专属约束
- restore/deploy 必须先 dry-run
- 不备份/传输 .env.secrets 明文
- 不覆盖远程已有 skills/ 内容
- prune 保留至少最近 1 个备份
- doctor 自动修复仅限于安全操作
- 分发包不包含 history.jsonl

### 工具层专属约束
- 不静默升级技能
- 卸载前检查反向依赖
- 安装来源验证
- 不破坏 skills 目录结构

## 高危操作确认

### 工具层操作
| 操作 | 确认要求 |
|------|----------|
| 安装技能（install） | 来源 URL + 技能名 + 版本 |
| 卸载技能（remove） | 技能名 + 反向依赖检查 |
| 更新技能（update） | 当前版本 → 目标版本 + diff |

### Ops 层操作
| 操作 | 确认要求 |
|------|----------|
| 全量备份（backup） | 备份范围清单 + 排除项 |
| 恢复备份（restore） | 备份文件路径 + dry-run |
| 打包分发（package） | 内容清单 |
| 远程部署（deploy） | 目标地址 + 传输清单 |
| 清理数据（prune） | 清理范围 + 保留天数 |

### 命令执行黑名单
| 命令 | 禁止原因 |
|------|----------|
| `rm -rf /` | 破坏性删除 |
| `dd` 直接写块设备 | 可能破坏磁盘 |
| `chmod -R 777` | 权限过于开放 |
| `> /dev/sda` | 设备破坏 |
| `curl ... | bash` | 远程执行 |

## 隐私保护

以下内容在输出和日志中必须自动脱敏：
- API Key、Token、密码、私钥
- 连接字符串中的凭据部分
- 个人身份信息（邮箱、电话、地址）
- 内网 IP / 非公开域名
- 芯片唯一 ID（UID）、MAC 地址
- 固件加密密钥、签名私钥

## 审计与异常检测

### Audit Log 记录项
| 审计项 | 记录内容 |
|--------|----------|
| 高危操作 | 操作类型 + 时间 + 确认状态 |
| 权限越界尝试 | 目标路径 + 拒绝原因 |
| 命令执行 | 完整命令 + exit code |
| API 调用 | 端点 + 方法 + 数据量 |
| 文件变更 | 路径 + 操作类型 + 变更大小 |

### 异常行为识别
- 高频高危操作：单位时间内超过阈值
- 权限试探：连续访问禁止路径
- 异常命令链：正常流程中不会出现的组合
- 信息外泄：向未知端点发送数据
- 规则漂移：越来越频繁跳过确认
